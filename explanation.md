# Optimization Explanation## Problems Identified1. No index on `customer_id` → slow JOIN2. No index on `order_date` → full table scan3. GROUP BY only on name (non-unique)---## Improvements Applied### 1. Indexing- Added index on `orders.customer_id`- Added index on `orders.order_date`### 2. Query Fix- Updated GROUP BY to include unique identifier (`c.id`)---## Results| Metric              | Before        | After         ||--------------------|--------------|--------------|| Execution Time     | ~10–12 sec   | ~150–200 ms  || Query Type         | Full Scan    | Indexed Scan || Performance Gain   | ~98% faster  |---## Key Takeaways- Proper indexing dramatically improves performance- Avoid grouping by non-unique columns- Always analyze execution plans for slow queries
